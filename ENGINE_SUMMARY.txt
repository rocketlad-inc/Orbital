KEY FEATURES
============

ATOMICITY
- All mutations happen in ONE transaction per tick
- If any operation fails, entire tick rolls back
- No partial state corruption possible

SEPARATION OF CONCERNS
- Engine logic separate from database
- Physics calculations isolated
- Data represented as immutable dataclasses
- Events logged to chronicle

TESTABILITY
- 31+ test cases with comprehensive coverage
- Mock database for testing without DB
- Error cases explicitly tested
- All operations documented with examples

EXTENSIBILITY
- Abstract Database interface (PostgreSQL, SQLite ready)
- OrbitalPhysics stub for custom implementation
- JSON-based standing order conditions/actions
- Chronicle events capture all changes

PERFORMANCE
- In-memory state eliminates N+1 queries
- Indexed database queries
- Batch inserts for chronicle
- Transaction isolation prevents conflicts

DOCUMENTATION

ARCHITECTURE
============

Order of Operations Per Tick:
1. Load game state from database
2. Execute maneuvers due this tick
3. Check for SOI transitions
4. Evaluate conditional standing orders
5. [Placeholder] Resolve combat
6. Accumulate resources for owned bodies
7. Advance technology research
8. Advance production queue
9. [Placeholder] Check treaty violations
10. Decay faction reputation

Data Model:
- OrbitElement: JSON-serializable orbital parameters
- Ship: Fleet unit with position and fuel
- Body: Celestial body with production
- ManeuverOrder: Queued fleet movement
- StandingOrder: Conditional automation rule
- ResourceInventory: Faction resources
- GameState: In-memory world snapshot
- ChronicleEntry: Historical event log

- tech_research: Technology progression

History:

All tables indexed for performance.

USAGE EXAMPLES
==============

Basic Tick Execution:
    engine = GameEngine(db)
    result = engine.execute_tick(game_id="game_1", tick=100)
    if result.success:
        print(f"Logged {result.eventsLogged} events")

Game Loop:
    for tick in range(max_ticks):
        result = engine.execute_tick(game_id, tick)
        if not result.success:
            break
        broadcast_events(result.chronicle)

Game Initialization:
    manager = GameManager(db_config)
    game_id = manager.create_game(
        name="Test Game",
        factions=[{"name": "Faction 1", "userId": "user_1"}]
    )
    manager.run_game_loop(game_id)
